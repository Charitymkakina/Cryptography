# Symmetric Cryptography
Is a mathematical operation that is performed with a secret key, verified or undone with the identical secret key
Symmetric encryption includes 3 operations:
   1. [Symmetric encryption](#1-symmetric-encryption)
   2. [Message Authentication Codes](#2-mac-hmac)
   3. [Pseudo Random Functions (PRFs)](#3-pseudo-randoms-functions-prfs)

# 1. Symmetric Encryption
Is a cryptographic process that transforms readable data (plaintext) into an unintelligible form called ciphertext using a secret key,
and later reverts the ciphertext into plaintext using the same key.
`Ciphertext` is the unreadable form of the message produced after encryption.
The purpose of symmetric encryption is to ensure that only parties who possess the secret key can read the original message, thereby providing confidentiality.
Advantage: 
  1. High efficiency- Very fast compared to asymmetric cryptography
  2. Suitable for encrypting large amounts of data (bulk data).

# 2. MAC-HMAC
`MAC` (Message Authentication Codes) is a cryptographic function that uses a secret key and a message to produce a fixed-size authentication tag
Hashing alone cannot provide authentication because anyone can compute a hash. A `MAC` solves this by requiring a shared secret key to generate the tag.
`MAC` ensures;
   1. `Integrity` - The message has not been altered
   2.  `Authentication`- The message was generated by someone who knows the secret key.

NOTE: `Hashing` does not use a secret key; anyone can generate it. MAC fixes this by adding a `secret key`, meaning an attacker cannot compute a valid `MAC` because they do not know the key

`HMAC` 
`HMAC` (Hash-based Message Authentication Code) is the most widely used MAC construction.
It uses a cryptographic `hash function` and a `secret key.` Example `HMAC-SHA256`
`HMAC` does a more complex double hashing process to prevent attacks.
Algorithms:
Modern: `HMAC` `Poly1305`
Future: `GCM` `CCM` `(AEAD Ciphers)`
